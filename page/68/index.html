零基础HTML编码

任务目的如下

  • 了解HTML的定义、概念、发展简史
  • 掌握常用HTML标签含义、用法
  • 能够基于设计稿来合理规划HTML文档
  • 理解语义话，合理使用HTML标签来构建页面

解答

1 HTML(Hyper Text Markup Language 超文本标记语言)最早是由为CERN工作的Tim Berners-Lee在1980提出的，为了方便搜索，需要通过使用标签对文档进行结构化定义，机器识别标签后就能知道文档的结构，文档应该按照结构来书写。比如<title></title>标签内部就应该写文章的标题。1995年产生HTML2.0标准，1997年1月发布HTML3.2。1997年10月发布HTML4.0，1999年10月发布HTML4.0.1,发布HTML4.0标准同时发布CSS，规定在CSS中描述html各个结构形状大小、颜色等，使得HTML文档结构和样式分离。2014年HTML5标准。

2 HTML规定标签种类、标签层次结构及标签属性，浏览器根据各种标签及属性按照规定的格式显示出页面。

html标签分为单标签<img />和对标签<body></body>，大多数标签都有属性，属性格式为属性名="属性值"，填写在起始标签后。除了上面有明显语义的标签外，还有功能性标签如<div></div>用于划分页面的区域

Ubuntu 搭建hexo博客和发布文章总结

发表于 2016-10-14 更新于 2017-06-16

看了scott的前端视频及相关文章，心血来潮打算搭建一个hexo博客来装一下，一方面可以练手创建静态网页各种技能，另一方面记录日常。原来的farbox考虑继续用还是迁移过来，还在犹豫中。
整个博客搭建基本参考金石开Hexo搭建Github静态博客，由于改用ubuntu系统了，整个过程还算顺利。遇到的问题主要有

1 markdown编辑器

本来打算使用sublime text配合markdown插件，编写hexo博客文章的，结果发现sublime对中文支持真的很不好，网上搜索发现retext编辑器用了一会确实很好，本文就是在retext编辑器中编写的。

2 github page项目的创建

一开始不明白github page项目和一般项目的区别，仔细浏览说明后才明白，每个github用户可以创建一个特殊的仓库，用于静态网页的展示。仓库名称必须为”github用户名”+”.github.io”,我的gitpage仓库名为”hhzzer.github.io”。以这种规则命名仓库后，github会自动识别项目为web展示页，这样我们就可以将生成的静态页面，托管到github网站上，然后以浏览网页的方式展示出来，而不是浏览仓库的模式。

3 绑定个人域名

个人博客有个独立域名，感觉是不是要牛逼一些呢，就开始折腾绑定个人域名。绑定个人域名达到的效果就是要通过blog.hhzzer.com 访问博客，而不是通过hhzzer.github.io访问。做域名跳转主要有2步:
第一，在自己的域名服务商网站添加一个C记录，指向要跳转的地址，即:blog.hhzzer.com–>hhzzer.github.io
第二，在gitpage仓库根目录添加CNAME文件，文件中写明blog.hhzzer.com,应该是接受该域名的跳转。
配置了上面两步后，独立域名能正常跳转了，但是每次新发布文章，都会把CNAME文件删除掉，网上找到方法是将CNAME文件放在hexo博客source目录。

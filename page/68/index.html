ife-task4

发表于 2016-10-29 更新于 2017-06-16

看完《CSS权威指南》，开始实践baidu-ife任务四。上一个任务是实现三栏式布局，中间栏宽度随浏览器窗口大小改变。实现方式是创建主div后包含依次创建的left、right、middle三个div，然后浮动left、right，middle元素通过外边距设置位置。主div高度自动取子元素最高通过设置overflow:auto;实现。

任务

这次任务目标：深入理解position等属性。实现效果链接

解答

此次练习主要考察position属性，根据要求需要实现方块在浏览器放大缩小时，方块始终在水平垂直居中位置。故position:fixed,确定窗口中间点，其他方块根据中间点位置做position:relative偏移。方块左上角和右下角有1/4圆覆盖，通过分别设置两个矩形方块背景设置为白色覆盖掉多余的3/4即可。

code review

看了此任务得分第一和得分第二,瞬间觉得自己实现的方法丑得掉渣。
得分第一的代码有三点设计很好

  • 只采用三个div，灰色背景盒子嵌套两个1/4圆，
  • 背景灰盒子定位采用absolute,对齐位置采用相对距离单位50%。偏移采用外边距设置。
  • 两个1/4圆角形状采用border-radius:0 0 50px 0方式，直接取得1/4圆。

得分第二的设计优点

  • 一个大盒子套两个小盒子方式，定位也是absolute.
  • 圆角采用扩大半径，overflow:hidden切掉多余部分。
  • 另外提供了采用table模型布局的方法。

我采用了找到中间点，然后其他元素相对于中间点布局的方式，和前两种方式优点为：缩放浏览器，不会产生滚动页面，图片始终对称排列。缺点为盒子使用太多，最终代码用了一个大盒子套3个小盒子，在html中改变div顺序，会打乱图形排列。浏览完榜上10个团队，发现定位方式有2类
1 只采用absolute
2 absolute+relative
只有我采用fixed+relative，难道是采用fixed的评分都不高？

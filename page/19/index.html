小富靠运，大富靠命

发表于 2022-07-04

发表于 2017-08-22
吴军老师这两天的来信，认真听了，能得到很多。

结论

小富靠运，大富靠命

证明

  • 为什么小富靠运气？

同门清华师姐高考运气好，本来上北京科技大学后来上了清华大学，运气好。后来的发展泯然众人，是命中注定。
美国通过中彩票获得大量才富的人的和在中国股市早期获得财富的探险家，后来都基本上败光，这也是运气好，命不好，无法大富。

  • 命的定义及获取财富的方法

每个人对命的定义不同，吴军老师的定义我比较赞同：命运是环境因素和个人看问题、做事方法绘制的两条线。获取财富的方法是信命、认命。
相信改变自己看问题、做事方式能改变命。如果由于长期行为累积出的习惯无法改变，就认命，不妄想。

  • 为什么大富靠运

Z 君看似经历一系列好运气，高盛、谷歌、亚马逊上市，但这都是由于他看问题、做事情方法给他带来的，命好是原因。出生只是运气，开始的好运气能让命好的概率大一点，但命运就像马拉松，塑造命运的过程才是跑完获胜的关键。
反证：李广难封，是由于命中注定。他又好运气可以上阵杀敌立军工，但都是已做士兵的心态行动。其实是不够资格封侯。

实践方法

练习对“扇耳光”的反应，有可能可以改变命运。三种反应：1）扇回去，2）认怂，3）分析该不该被扇，该就改，不该就给对方教训。
日常的谈吐可以看出一个人的命运，就是看他怎么反应，举例袁腾飞判断他学生妈下岗。
三同学 A，B，C 聚会，分出三类人。A 官员，被扇后思考，加入强势阶级，有权扇人。B 教授抱怨认怂，不认命也不反抗，必然结果。C 学者，思考后强大自己，让人无法扇到。

我的思考

以合作者姿态做事，与信命者共事，自私的自己把握命运。

互联网文明怎么改变城市-精品课读后感

发表于 2022-07-04

发表于 2017-08-16
昨天花 9.9 买了得到的一个精品课，今天回顾一下主要内容和感受。

课程概述

课程 10 小节，每一节回答一个问题。

  1. 城市为什么会繁荣？
    因为城市造就了高密度的人际空间，使人与人之间协作效率提高，创造价值。所以城市会打败乡村。
  2. 城市为什么会衰败？
    城市的价值分为信息价值和资源价值，信息价值逐渐被互联网取代，只保留资源价值的城市会衰败。互联网蓬勃发展的城市由于公司给城市注入了互联网从其他城市吸取的信息价值，所以还会发展。买房就是买城市的股票，现在是时候买科技公司股票了。
    城市资源价值会不会被移动车辆取代？驭势的客厅型车辆
  3. 互联网进化快于城市，所以趋势不会变化
    深圳是城市迭代速度的极致，30 年成就今天。互联网由 pc-移动端-物联网迭代数度指数增长，城市进化肯定比不过互联网。
  4. 为什么我们会选择留在环境不好的城市
    城市建筑的非实用性功能会刺激产生多巴胺，让人愉悦。但互联网更能刺激你，使你对城市的低强度刺激免疫。
  5. 标准的城市规划《雅典宪章》为什么失效
    雅典宪章是指城市按照功能分区规划，工作区、生活区、休闲区、交通区。在伦敦改造时成功，在巴西利亚、西安建设时失败。原因在于城市是生长出来的不是计划出来的，和市场经济活力强于计划经济一个道理。
  6. 互联网会引导城市生长得很好
    互联网取代城市的信息价值，信息检索也由具体地理位置变成搜索关键字。互联网发现城市现有资源的价值
  7. 为什么快递、外卖、便利店杀死城市？
    快递、外卖、便利店让城市资源由集中转为分散。原因一时互联网检索改变城市位置信息发布，二是快递、外卖、便利店让资源围绕着人配置
  8. 城市自动生长，人可以起发挥作用
    人通过借助互联网发现城市价值，实现自己价值。airbnb
  9. 公共空间微型化
    人需要交流，公共空间满足交流需求。
  10. 未来城市是怎么样的？
    建筑是居住的机器。随身城市、大脑城市，是终点。

我的想法

买房就买北深杭，买股就买 JAT，企业周围，大有作为

python创建zabbix主机导入xml信息

发表于 2022-07-04

发表于 2017-08-14
使用 python xml.etree.ElementTree 模块创建可导入 zabbix 的主机列表文档，加快监控添加效率。

问题来源

工作中需要批量添加大量主机到 zabbix 监控。虽然 zabbix 自动发现功能自动添加，但是添加的无描述信息，监控组无法通过告警信息快速联系责任人，所以监控主机需要手工添加。zabbix 系统中添加监控信息的途径有：1）自动发现添加 2）web 界面手动一台一台添加 3）编辑 xml 文件，批量导入 4）使用 api 导入。本文完成第三种模式的添加。

IOP（input、process、output）分析

创建 xml 文件有两种方法：一是引用一个 xml 模板，解析其中的结构，然后复用解析的元素;二是从零生成一个满足导入需求的 xml 文件。第一种操作快，但涉及的流程复杂，第二种为笨办法，但流程清晰，故采用从零创建。

  • input
    input 为一个 ip 列表，可以是一个范围
  • process
    分析了 zabbix 导出的 xml 结构，发现主机信息在 xml 文件中部子节点中。故先创建主 xml 结构，然后通过循环创建子节点。子节点创建定义一个函数方便调用。
  • output
    输出为一个 xml 文件

遇到的问题

  1. 导入的 xml 模块需要使用 python2.7 环境，python3.5 环境报错
  2. “Invalid tag “/zabbix_export/version”: ”发现 version 版本写的“3”，而模板是“3.0”,改为 3.0 后问题解决
  3. 中文编码问题，生成 xml 文件中有中文字符会报错。程序运行为 ubuntu，python 文件编码为 utf-8,python 内部编码为 unicode。应该把中文字符转换为 unicode 后能解决。解决办法：python 文件采用 utf-8,在文件头添加“#-- coding: UTF-8 --”解决“None-ASSIC”问题，中文字符采用”u’中文’”模式，解决“UnicodeDecodeError”
  4. port 标签忘赋值，添加后导入不报错。

后续改进

ansible 上有添加 zabbix 监控主机的模块，应该使用 api 实现的。后期要么使用 ansible 的 zabbix 模板，要么自己封装 api。

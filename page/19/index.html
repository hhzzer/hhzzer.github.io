读书笔记写法摘录

发表于 2022-07-04

发表于 2017-08-03
吴军推荐了一堆书，读的时候想着留下点什么记录时，才会更投入去读。读书笔记就是强制记录的手段，别人怎么看待读书笔记和怎么记录值得我参考

费曼学习法

学习新知识最好的办法就是用，如果没有机会用，就去教别人。

王烁记法

读书笔记记录读的时候受到的启发，触类旁通的感想，以及待探索的线索。一本书的核心关键已经在导言、结论两章写明。如果真要自己抓关键，可通过：抓主题、列大纲、层层分解。

万维钢强力研读法

1.清晰表现每一章的逻辑脉络

我习惯完全按照原书的章节给读书笔记划分章节，甚至保留各章节的标题。不过这种内容提要并不重要。重要的是一定要看出来作者的逻辑脉络。每一章的逻辑结构如果真写出来也许只有几句话，可是这几句话却常常是分布在好几十页中。

2.带走书中所有的亮点

好的读书笔记是不均匀分布的。笔记，是我听说了一个想法之后很激动，必须把这个想法记下来具有已有的行为。所以逻辑脉络之外，如果发现真正好的小故事–我们称之为“亮点”–那么就把这个故事也给写下来，甚至具体到细节。

3.大量自己的看法和心得

强力研读是一种主动式的读书。要在笔记中写下自己对此书的评论，好像跟作者对话一样。

4.发现这本书和以前读过的其他书或文章的联系

当你度过的数多到一定程度，你就会发现书与数之间是存在练习的。如果你读过足够多，你就会获得一种更难得的经历：感受人类知识的进步。你会发现一个问题在这本书里是这个说法，而过了几年之后有人写了另一本书，引用更有力的证据，把整个结论给改变了。
笔记是对一本书最大的敬意。读书笔记是一种非常个性化的写作，是个人知识的延伸。如果做不到强力研读的笔记标准，随便做一个一般水平的读书笔记对自己也有帮助，起码能加深记忆力。

我对这篇文章的笔记？

读书笔记两大内容：1）记录书讲了什么，通过什么样的逻辑讲的。2）我对书中观点的看法，赞成、否定、疑问、观点修正？
如果再深入记录就是带走亮点，整理自己整个认知体系。

定制化centos6安装光盘遇到的坑及解决过程

发表于 2022-07-04

发表于 2017-08-02
隔一段时间就需要安装大量物理机的操作系统，安装过程中设置时区、语言、键盘、分区、选软件包，安装后还需要禁用 seliux、firewall 等。这些费时操作能否通过参数文件指定，然后加快安装速度？于是就开始可定制化安装光盘踩坑之旅。
由于安装环境网络隔离，无法使用网络启动安装系统。物理距离远，也无法抱笔记本作为网络启动服务器，所以定制化安装光盘是唯一的出路。

解决过程

ks.cfg

使用 ultraiso 修改了 iso 镜像后，就无法安装系统

测试使用了“centos6.9minimal”和“centos-7-x86_64-Minimal-1611”两个版本的 iso，使用 ultraiso 编辑 iso 文件，添加任意文件到 iso 并保存，采用编辑后的文件都无法安装系统，都报错”Unable to read group information from repositories” 。之前一直以为是编写的 ks.cfg 文件参数错误，直到看到这篇文章才发现，是 iso 文件错误。解决办法是重新生成 repo，重新生成 iso。

总结

之前一直使用 ultraiso 编辑镜像后保存安装系统一直报错，有些怀疑是编辑镜像的原因，排查到 yumrepo 报错时，终于顿悟。
目前只做了 centos6.9minimal 的安装 iso，根据情况制作 centos7.2 的。ubuntu 的定制化再找时间测试。

配置工作用centos7系统

发表于 2022-07-04

发表于 2017-08-02
工作网络和自带设备网络隔离，需要使用堡垒机。堡垒机为 windows 系统，很多工具在 linux 下使用方面，故需要早 windows 机器上创建安装一个 linux 工作环境。

操作环境配置

  1. 调整虚拟机版本为 9,并修改配置
    堡垒机为 vmware 虚拟化的虚拟机，版本为 8.要在虚拟机中嵌套虚拟机，需要把虚拟机版本升级为 9 及以上，并在 web 管理界面开启虚拟化支持。参考链接
  2. 安装 vmware workstation
    下载链接
  3. 配置虚拟机资源 1C1G10GB,网络选用 VMnet8,配置端口映射：内部 22-外部 2222。
  4. 安装 centos7 系统。时区 Aisa/Shanghai basic .安装后无法启动图形界面
  5. 设置 yum 代理 vi /etc/yum.cfg “proxy=http://192.168.x.x:port"
  6. 安装软件 yum tree ansible r

ife-task4

发表于 2016-10-29 更新于 2017-06-16

看完《CSS权威指南》，开始实践baidu-ife任务四。上一个任务是实现三栏式布局，中间栏宽度随浏览器窗口大小改变。实现方式是创建主div后包含依次创建的left、right、middle三个div，然后浮动left、right，middle元素通过外边距设置位置。主div高度自动取子元素最高通过设置overflow:auto;实现。

任务

这次任务目标：深入理解position等属性。实现效果链接

解答

此次练习主要考察position属性，根据要求需要实现方块在浏览器放大缩小时，方块始终在水平垂直居中位置。故position:fixed,确定窗口中间点，其他方块根据中间点位置做position:relative偏移。方块左上角和右下角有1/4圆覆盖，通过分别设置两个矩形方块背景设置为白色覆盖掉多余的3/4即可。

code review

看了此任务得分第一和得分第二,瞬间觉得自己实现的方法丑得掉渣。
得分第一的代码有三点设计很好

  • 只采用三个div，灰色背景盒子嵌套两个1/4圆，
  • 背景灰盒子定位采用absolute,对齐位置采用相对距离单位50%。偏移采用外边距设置。
  • 两个1/4圆角形状采用border-radius:0 0 50px 0方式，直接取得1/4圆。

得分第二的设计优点

  • 一个大盒子套两个小盒子方式，定位也是absolute.
  • 圆角采用扩大半径，overflow:hidden切掉多余部分。
  • 另外提供了采用table模型布局的方法。

我采用了找到中间点，然后其他元素相对于中间点布局的方式，和前两种方式优点为：缩放浏览器，不会产生滚动页面，图片始终对称排列。缺点为盒子使用太多，最终代码用了一个大盒子套3个小盒子，在html中改变div顺序，会打乱图形排列。浏览完榜上10个团队，发现定位方式有2类
1 只采用absolute
2 absolute+relative
只有我采用fixed+relative，难道是采用fixed的评分都不高？

设计稿HTML架构解析
设计稿是由页面头部header、页面主体和底部栏组成。header部分由logo图标、4个导航链接及黑色背景组成，导航链接通过ul-li嵌套a构成。主体部分由三篇文章

版权文字和背景色组成。

CSS(Cascading Style sheets 层叠样式表)用于告知浏览器如何将文档呈现给用户。最初netscape和IE两种浏览器将各自实现的标签添加到html规范中，导致文档在不同浏览器显示不同。为在不同浏览器显示相同效果，将html规范中关于效果显示的特性独立出来，形成CSS标准。CSS标准在1997年随者HTML4.0标准一起发布。
HTML元素可以被多个描述信息定义，其中优先级关系为:内联样式(HTML元素内部style属性)>内部样式(HTML 文档header部分)>外部样式(外部引用.css文件)>浏览器默认样式。CSS语句为选择器+属性申明构成。用,将选择器分组，同一申明可应用在多个元素上。所有HTML子元素都继承父元素样式，子元素单独定义样式优先级高于继承样式。通过多个标签定位一个元素，称为派生，这样能简洁的选择到需要的元素。采用#id形式来使用id选择器，id值在HTML标签中通过id属性设置。采用.class形式使用类选择器。采用[title]形式选择带title属性的元素，属性选择器可通过~ | ^ $ *实现包含、单词匹配、开头字符匹配、结束字符匹配、任意位子字符匹配。元素大小、元素内边距padding、元素边框border、元素外边距margin依次由内到外展开。元素大小的百分比参数是参照父级元素定义的。div、h1、p为块级标签，strong，span为行级标签。

总结

自己以前没有认真学习CSS，觉得CSS很简单，本次任务实践过程中，才发现并非如此。特别是布局方式float,flex对padding、martin的影响。在查看别人代码过程中发现了HTML更多的语义化标签。figure->figcaption 用于标记带描述的图片。后面需要看书系统化学习HTML标签和CSS盒模型。

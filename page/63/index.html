开始写作

发表于 2017-04-18 更新于 2017-06-16

自己挣扎写作好久，今天听了古典的写作方法和李笑来的写作心法，那就开始写吧。

古典五问写作方法

  1. 找到一个强烈想表达的观点
  2. 这个观点和其他观点的不同
  3. 3个子概念支撑观点
  4. 观点带来的可执行方法
  5. 你对观点的看法

李笑来写作心法

认识到世界的不完美和自己的不完美，马上开始去做，在做的过程中不断修正认识。

示例

工作是你出卖时间获得收益的一个途径。其他观点认为工作是融入社会的一个方式，我认为其实就是一种交易。自己的资本就是时间，在单位时间内，凭借自己的技能或能力能解决别人的需求，这种满足需求能力的稀缺性决定了你的单位时间价格。可执行方法就是做到行业第一，增加自己稀缺性。不断学习东西，触类旁通的学习，不管在目前看来是有用还是无用。我的观点是工作是交易，我是商人，要获得财富就要专业化，专业化提高生产效率，创造更多财富。

个人收获总结与市场分析

发表于 2017-03-23 更新于 2017-06-16

个人博客就是一个仓库，放心的写吧，不暴露个人隐私，不泄露公司隐私即可。简书或微信公众号是展台，挑选精品展示。

最近休假，把自己拿去市场上试了一下水，这篇文章就复盘一下过程和一些想法。

自己做了好几年企业运维，想直接转行转工作内容太困难。公司需要直接上手的熟练工，个人也舍不得放下职业积累从零开始。先分析一下自己积累再分析市场上企业需求。

个人积累

第一年做了桌面服务台和弱电建设方面的工作。

办公环境的维护主要需要问题处理能力、沟通能力。问题处理能力需要在实践中摸索并且 总结记录。找方法主要通过内部案例和google搜索，解决问题过程则是控制变量法，所以问题解决率还是比较高。每次问题的总结记录欠缺，找不到合适的记录和传播方式。当前记录方式目的是基于用户的工单，便于考核维护人员。理想的资料库是为了快速解决问题和优化系统避免再次发生。问题记录三段回顾一下：1)问题描述，描述问题发生在什么地方、第一次出现还是持续出现，问题出现结果。2)问题复现，验证问题。描述清楚问题复现的过程。3)初步猜测问题原因，1,2,3并逐个验证，要控制每次验证的变量。4)总结整个过程，得出验证后的方法及可改善途径。沟通能力就是要站在对方立场帮助他解决问题，不能在你的立场上看待问题。一个在你看来是一个很白痴的问题，插一下网线就能解决的，你要先认同他的情绪，让用户感受到我理解他，出现故障确实让人心烦，然后再快速排查解决。解决后还要告之大概原因和使用注意事项或长久解决方法。

弱电项目建设主要是工程方面事情。自己首先了解各种布线方式，有哪些设备组成，优缺点是什么。然后从施工难度、材料成本、稳定性、扩展性及可维护性几个方面组合成一个适合的方案。验收过程最好有具体的电汽参数标准，次之是接上设备按效果验收，最次为测线仪测。

第2~3年开始做系统集成相关工作。视频监控、门禁安防、一卡通扩建和维护，有技术的二次开发，也有商务的维保立项。交换机、防火墙、负载均衡、入侵检测说是都做了，但没有融汇贯通。交换机、防火墙方面还较为熟悉，但也缺乏总结。

第4~今，开始做非业务系统的维护，业务系统需求变更及一些系统建设项目。维护了终端安全管理系统、文件服务器等，偏管理。在安全管理和满意服务之间平衡是较大的挑战。业务需求变更，是了解熟悉业务的一个途径，有时后觉得业务部门都不能讲明白业务，但要知道他们是熟悉明白当前业务的，需要挖讲出来内容后面的东西。《人人都是产品经理》中讲到的需求四象限：定性<–>定量，说<–>做。这个需要长期练习和总结，才能提高，我目前还是被业务说的牵着走。这方面的技能是做产品的核心技能，未来看涨，要采用科学方法、投入时间强化训练。

市场需求

运维分为桌面运维、产品运维、开发运维等。每种企业类型，根据主营业务需求不一样。有电脑和电子设备，就需要桌面运维，依据规模确定运维量再看是兼职还是全职。桌面运维可替代性高，单位时间价值不大。产品运维主要做业务系统按摩，监控系统状态、处理系统故障、优化系统。现在产品迭代快，实行敏捷开发，一边开发一边运维就导致了开发运维，这类能理解需求、做出产品、上线运行的岗位能学习到最多的知识，个人价值成长快。另外还是信息基础设施运维的，机房服务器运维、云服务器运维，长期和机器打交道，熟悉自动化工具。

企业类型有传统行业、系统集成商、运营商及设备商外包、软件开发商、互联网产品公司。传统企业需要桌面运维，规模较大的需要机房运维、产品运维等，运维在企业中起辅助支撑作用。系统集成商需要弱电技术方案人员和实施维护人员，项目中若包含软件，还需要软件二次开发。运营商及其外包公司需要运营其软件及网络设备。

用商业视角看行业，用行业视角看企业，用企业视角看个人。降维观察，每个人都是一个企业。

寻得一人，共度一生，之后更需考虑要走什么样的路。

生活的目的

我们既非自愿也无人邀请，就到这世上做一短暂停留，目的是什么？
爱因斯坦说我们是为他人而活着，因为我们的生活也依赖与他人。吴军说人生最终的目的，是生活。我们学习、工作、寻找伴侣都是为了走完这一短暂旅程。古典说一生就是事业、家庭、个人三方面的平衡。你选择困难模式，拼搏进取，收获跌宕起伏的体验也好；选择简单模式，随遇而安，听从社会的安排也罢。这都需要你用整个生命时间来支付。人的天性是向往自由的，就像马本能的喜欢在草原上奔跑一样。自由就是选择题的选项，选项的多少有很大概率由钱的多少决定，因此要自由首先要财务自由。

人人都是产品经理-读书笔记

发表于 2017-03-20 更新于 2017-06-16

极其功利的阅读此书。工作上开始做项目，遇到疑惑学习一下别人的理解。

产品经理：做正确的事，并把事情做成。项目经理： 在规定的时间内把事情做成。

甲方项目经理（IT）：理解业务需求，协调开发资源，根据自己的技术背景选择合适的系统构建方式。
甲方项目经理（业务）：表明项目业务价值，明确业务需求。
乙方项目经理：理解需求、确认需求、分解开发、完成上线。

产品经理做产品就像生孩子并养育，需要明确为什么生、生孩子带来的价值、生孩子的投入、养孩子的投入。项目经理就是助产护士，知道孩子要在什么时候生出来，做生孩子前期准备并保证生的过程顺利，生完拍屁股走人。

公司网络架构思考

发表于 2017-03-20 更新于 2017-06-16

局域网架构物理上分为三层，接入层、汇聚层、路由层。路由层由两台7600通过vss（虚拟交换系统）功能，虚拟为一台设备，一主一备。配置需要在两边分别配置，不会自动同步。其中一台故障后，所有流量都自动切换到另一台。

防火墙划分为三个区域，lan，wan，dmz。通过静态路由，写明各个接口的下一跳，确保联通。dmz区域接应用负载，通过vlan隔离。wan口接链路负载，用于多运营商接入。

服务器分布在dmz区域和lan区域，对内外都提供访问的放在dmz区域。服务器硬件类型有ibm aix，ibm x86,联想、惠普、戴尔。aix日志设置与查看

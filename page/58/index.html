腾讯蓝鲸运维平台初探

发表于 2017-05-27 更新于 2017-06-16

一入运维深似海，从此床上不折腾。凌晨3点的短信电话告警、无休止的变更，让朝气蓬勃的青年变了样。网传蓝鲸平台标致可人，今日亲测之。

基本概念

蓝鲸运维平台3.0需要内测申请，每周2,4审批，故测试2.0版本。

运维平台也叫工作台，在工作台中可添加系统默认应用和自己编写应用。系统默认应用包括CMDB、job平台。为了便于自己编写应用，工作台中有开发者中心标签，实际为pass平台，用于快速创建在工作台上的应用。rabbitMQ用于异步消息传输，用于应用间消息传递、作业平台命令传递到主机。

部署

环境准备

使用两台服务器部署，1台部署基础模块+正式应用+MQ，1台部署app测试环境。操作系统版本为centos6.5_X64（使用centos7.2由于内网无法更新部分软件包，遂放弃。）

安装好系统，配置好网络和yum源后，就可以下载证书和软件了。注册平台帐号，在证书下载页面输入mac地址，下载证书。在社区版下载下载v2.0.1版本（需登录）。

安装步骤

排错处理

  1. 安装后报证书错误
    输入正确的mac地址后再下载证书，init后，要执行install。
  2. 安装pass正式环境报错找不到register.tmp.
    需要将基础模块启动后再安装，参考.
  3. 安装rabbitmq时无法启动报错“erl：command not found”
    表明环境变量设置有问题，先whereis erl找到erl所造目录，然后在～/.bash_profile中添加，然后执行source ~/.bash_profile重新加载
  4. rabbitmq无法启动，在平台上无法显示
    在第三方服务中添加第三方服务，输入ip地址、用户名和密码。

跟踪经济的思考

发表于 2017-05-25 更新于 2017-06-16

物联网的本质是跟踪，充分利用工作的环境，理解物联网，挣大钱。

吴军在《硅谷来信》中讲了未来经济的特点共享经济、跟踪经济、合作经济、众筹经济。跟踪经济分两天讲，重点来理解下。

物联网的到来，让我们对人和物的跟踪方便便宜。传感器收集信息方便。

机器智能使得分析跟踪产生的大量数据，抽象出模型，产生价值花费减少。

跟踪的目的是更详细的了解跟踪对象，对象包括人和物品。

对物品的深入了解有助于物品的生产者、销售者和消费者。制造者了解物品的生产过程、流通过程和消费用户，能获得直接的反馈信息，用于改善设计、质量。销售者获取物品上下游信息历史数据有助于精准营销。消费者了解物品渠道和生产信息能减少购买假货、劣质品。

对人的跟踪包括对行为的跟踪和对健康数据的跟踪。行为数据通过模型分析可以推测健康数据。个人对于自我的跟踪目的也是更好的了解自己，改善自我效率、目标、价值、健康等。

跟踪经济的核心是人。第一代通过邮箱或帐号作为识别ID，用cookies跟踪个体的上网行为，投其所好，引导消费。第二代用手机号作为识别ID，手机、电脑多方位收集支付信息、位置信息、交流信息更多维度刻画用户。第三代利用可联网的传感器从外部收集人的信息，各种维度，能更准确的识别人，滴滴出行、共享单车、支付宝。

腾讯通过发身份证，再引导用户使用各种网络服务、社交活动，了解人。阿里通过网络行为、线下团购、出行等特征，合成一个人。最终两家公司会完全了解你，你的性格、喜好、收入、工作，完全掌握，然后把你玩弄于手掌之中。

家庭摄像头产生原因是老龄化、二娃及对安全缺失的恐惧。智能手环收集运动信息并做朋友排名，满足健康和虚荣需求。智能跑鞋和运动手表用于收集和分析运动数据，优化训练。儿童手表跟踪位置和主动确定行为，解决父母对儿童安全的恐惧。

ansible-playbook学习1

发表于 2017-05-23 更新于 2017-06-16

同事写了几个ansible playbook，借鉴学习一下。一个是升级ssh，另外两个是安装zabbix_agentd.

升级ssh

升级ssh需要升级openssl和openssh，升级方式都是采用源码安装。
playbook使用了shell模块，shell模块是在远端开启一个/bin/sh运行命令，与日常ssh到服务操作命令一致。command模块不能使用变量和操作符（$ | <），可以使用who,不能使用ulimit -a。sudo: yes表示执行脚本使用sudo执行，在playbook外部参数中使用-K，手动输入sudo密码。

安装zabbix_agentd

此playbook hosts 直接写地址，就不用在/etc/ansbible/hosts中定义？
使用模块依然是shell，shell中用到了sed命令，用于修改配置文件sed -i 's/Server=127.0.0.1/Server=192.168.x.x/g' /etc/zabbix/zabbix_agentd.conf 。使用ansible提前获取的系统信息变量{{ ansible_ens160["ipv4"]["address"] }}赋值到配置。
针对centos7和centos6因为启动服务命令不同写了两个playbook，后面也可根据setup信息做条件判断。

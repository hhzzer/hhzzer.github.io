再记录源码安装zabbix-agent

发表于 2022-07-04

发表于 2017-06-26
yum 安装 zabbix 虽然方便，但是不同硬件架构系统要不同的 rpm 包，寻找麻烦。源码安装只要有源码和 gcc 编译器，就可以编译出适合不同硬件架构不同系统的 agent。今天要在 IBM power8 上安装 agent，再次记录一下，并修正以前的错误理解。

安装步骤

1 下载源码，版本 3.0.9
2 复制源码到目标机器
3 解压并编译,编译后默认可执行文件所在目录为”/usr/local/sbin/zabbix-agentd”

4 编写配置文件。为了与 yum 安装的 agent 目录保持一致，配置文件目录为”/etc/zabbix/zabbix_agentd.conf”

5 验证。
sudo /usr/local/sbin/zabbix_agentd -c /etc/zabbix/zabbix_agentd.conf ,ps -ef | grep zabbix*,就可以看到 5 个 zabbix 进程。
6 安全
zabbix_agentd 默认以 zabbix 用户启动，若没有 zabbix 用户，则以执行启动命令用户启动。为保证权限控制，建议创建 zabbix 组和用户

7 启动优化
每次启动时都需要输入完整执行文件路径和配置文件，停止需要 kill5 个进程，比较麻烦。将启动命令做成服务，就可以用 service zabbix-agent start|restart|stop 管理。设置方法为在/etc/init.d/ 中创建服务脚本,chkconfig zabbix-agent on 设置开机启动。

启动有权限问题，修改 log 文件权限为 777

https://serverfault.com/questions/29788/what-is-needed-for-a-linux-service-to-be-supported-by-chkconfig

lineinfile 模块示例

  1. 删除文件中的行，删除 test 文件中所有匹配 hello 行。改变正则表达式可以改变匹配条件。“regexp=’^hello’”,以 hello 开头的行。

YUM基本概念及配置解释

发表于 2022-07-04

yum 经常使用，探究一下 yum 使用的相关问题及为什么。

概念

rpm： RedHat Package Manager,将源码编译并记录相关依赖的安装包。不同发行版本的 rpm 包不同。
yum：Your Update Manager，自动处理 rpm 包依赖关系，制作为依赖表，自动安装指定软件。
Repo：yum 的软件仓库配置，一般在/etc/yum.repo/下，每一个配置文件可包含多个软件仓库。yum repolist all ,列出本机所有软件仓库。
yum 软件库：在线或本地的 rpm 包目录，包含入 rpm 包和 repodata（软件依赖数据库）。在 repo 文件中配置。

yum 软件仓库配置文件解读

cat /etc/yum.repo/Centos-Base.repo

  • [base]: 软件仓库id，所有repo中应该唯一。
  • name:仓库别名，方便人为识别。
  • mirrorlist:仓库镜像链接，全球多个镜像，配合插件可从最快镜像中获取软件包。
  • baseurl:软件仓库实际地址，支持 http，ftp，file:///
  • enabled: 软讲仓库开关，1,开启，0 关闭。默认 enabled=1
  • gpgcheck: 是否检查 rpm 包内的签章，1 检查，0 不检查。用于验证软件包是否为发行厂商分发。
  • gpgkey： 数字签名公钥位置。

$releasever 变量值为yum配置文件/etc/yum.conf中distroverpkg行=后面软件包发行版本。执行rpm -q –qf %{version} centos-release;echo可获得。
$basearch ,系统是 32 位还是 64 位 运行 arch 获得值。url 中采用变量方面不同版本不同位数系统使用同一个 repo。

  1. 内网服务器怎么安装和更新软件包？

如果内网搭建了无密码的 http 代理服务器，可修改/etc/yum.conf,在末尾添加 proxy=http://ip:port
如果没有搭建代理服务器，可使用局域网 yum 源和本地光盘 yum 源。

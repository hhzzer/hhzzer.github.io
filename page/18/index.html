互联网文明怎么改变城市-精品课读后感

发表于 2022-07-04

发表于 2017-08-16
昨天花 9.9 买了得到的一个精品课，今天回顾一下主要内容和感受。

课程概述

课程 10 小节，每一节回答一个问题。

  1. 城市为什么会繁荣？
    因为城市造就了高密度的人际空间，使人与人之间协作效率提高，创造价值。所以城市会打败乡村。
  2. 城市为什么会衰败？
    城市的价值分为信息价值和资源价值，信息价值逐渐被互联网取代，只保留资源价值的城市会衰败。互联网蓬勃发展的城市由于公司给城市注入了互联网从其他城市吸取的信息价值，所以还会发展。买房就是买城市的股票，现在是时候买科技公司股票了。
    城市资源价值会不会被移动车辆取代？驭势的客厅型车辆
  3. 互联网进化快于城市，所以趋势不会变化
    深圳是城市迭代速度的极致，30 年成就今天。互联网由 pc-移动端-物联网迭代数度指数增长，城市进化肯定比不过互联网。
  4. 为什么我们会选择留在环境不好的城市
    城市建筑的非实用性功能会刺激产生多巴胺，让人愉悦。但互联网更能刺激你，使你对城市的低强度刺激免疫。
  5. 标准的城市规划《雅典宪章》为什么失效
    雅典宪章是指城市按照功能分区规划，工作区、生活区、休闲区、交通区。在伦敦改造时成功，在巴西利亚、西安建设时失败。原因在于城市是生长出来的不是计划出来的，和市场经济活力强于计划经济一个道理。
  6. 互联网会引导城市生长得很好
    互联网取代城市的信息价值，信息检索也由具体地理位置变成搜索关键字。互联网发现城市现有资源的价值
  7. 为什么快递、外卖、便利店杀死城市？
    快递、外卖、便利店让城市资源由集中转为分散。原因一时互联网检索改变城市位置信息发布，二是快递、外卖、便利店让资源围绕着人配置
  8. 城市自动生长，人可以起发挥作用
    人通过借助互联网发现城市价值，实现自己价值。airbnb
  9. 公共空间微型化
    人需要交流，公共空间满足交流需求。
  10. 未来城市是怎么样的？
    建筑是居住的机器。随身城市、大脑城市，是终点。

我的想法

买房就买北深杭，买股就买 JAT，企业周围，大有作为

python创建zabbix主机导入xml信息

发表于 2022-07-04

发表于 2017-08-14
使用 python xml.etree.ElementTree 模块创建可导入 zabbix 的主机列表文档，加快监控添加效率。

问题来源

工作中需要批量添加大量主机到 zabbix 监控。虽然 zabbix 自动发现功能自动添加，但是添加的无描述信息，监控组无法通过告警信息快速联系责任人，所以监控主机需要手工添加。zabbix 系统中添加监控信息的途径有：1）自动发现添加 2）web 界面手动一台一台添加 3）编辑 xml 文件，批量导入 4）使用 api 导入。本文完成第三种模式的添加。

IOP（input、process、output）分析

创建 xml 文件有两种方法：一是引用一个 xml 模板，解析其中的结构，然后复用解析的元素;二是从零生成一个满足导入需求的 xml 文件。第一种操作快，但涉及的流程复杂，第二种为笨办法，但流程清晰，故采用从零创建。

  • input
    input 为一个 ip 列表，可以是一个范围
  • process
    分析了 zabbix 导出的 xml 结构，发现主机信息在 xml 文件中部子节点中。故先创建主 xml 结构，然后通过循环创建子节点。子节点创建定义一个函数方便调用。
  • output
    输出为一个 xml 文件

遇到的问题

  1. 导入的 xml 模块需要使用 python2.7 环境，python3.5 环境报错
  2. “Invalid tag “/zabbix_export/version”: ”发现 version 版本写的“3”，而模板是“3.0”,改为 3.0 后问题解决
  3. 中文编码问题，生成 xml 文件中有中文字符会报错。程序运行为 ubuntu，python 文件编码为 utf-8,python 内部编码为 unicode。应该把中文字符转换为 unicode 后能解决。解决办法：python 文件采用 utf-8,在文件头添加“#-- coding: UTF-8 --”解决“None-ASSIC”问题，中文字符采用”u’中文’”模式，解决“UnicodeDecodeError”
  4. port 标签忘赋值，添加后导入不报错。

后续改进

ansible 上有添加 zabbix 监控主机的模块，应该使用 api 实现的。后期要么使用 ansible 的 zabbix 模板，要么自己封装 api。

为什么牛人离开大型技术公司-翻译

发表于 2022-07-04

发表于 2017-08-09
今天在阮一峰微博上看到赞赏这篇文章，我就尝试翻译一下。原文链接
我的一个学生带领我参观了一个大型上市技术公司，他是这公司的 CFO。这家公司仍热是最热门的技术公司之一。他们使得硬件在嵌入式软件和服务中大规模应用。
CFO 邀请我留下来，因为一个技术负责人要来参加一个会议。
我真希望我没去做…
负责人在那里抗议，因为他所有 70 人的团队需要从 Palo Alto 到东海岸。”现在团队大部分人步行上班，以后或许是乘火车上班。这让他们在路上会有 45 分钟休息时间。团队大部分人将离开我们。“
负责人向他的老板抱怨过，技术总裁插着手说这是“设备问题”，设备总裁把这个问题汇报给了 CFO。所以，组织一个会议是最后的手段，技术负责人为让他的团队留在城里向 CFO 做最后的汇报。
虽然这家技术公司的大多数人是从事制造业，然而决策小组是由经验丰富的软件工程师组成。”鉴于他们也可以在本地咖啡店的工作”，CFO 的回复让我目瞪口呆。”太糟糕了，但是我们需要空间。他们在这里工作已经够幸运了。至少他们在简历上留下了我们公司的名字”
什么？我不确定是我还是技术负责人更震惊。
这个负责人走了以后，我必须看起来很惊讶的样子，CFO 解释说：”我们有成千上万名雇员，在湾区，我们的空间需求根本不可能匹配我们的增速。你了解我们 CEO 的，‘喜欢我们或离开’是他一直观点。”（巧合的是，这个 CEO 在 20 年前是我创业公司的实习生。）我问到：”现在公司已经上市并且增长巨大，他的观点改了么？”CFO 回答道：”没有，我们的 CEO 相信我们正处于改变世界的巨大机遇中，只有在这里工作才能赶上，要么享受要么离开。我们依然收到大量想加入我们公司的简历，所以他认为没有必要改。”
我不知道该改成什么，想想这个观点可能有用，毕竟让一个初创公司成长为一个有 1 万或更多雇员的公司。”我们在改变世界的巨大机遇中，要么加入公司改变世界，要么滚蛋“这和我当时在当实习 CEO 的信念一样。

成人监督

在独角兽快迅速崛起之前，董事会仍然能掌控时，在他们找到合适的产品或市场后，他们”鼓励”创始人雇佣”家长型”人事管理公司。
因此多数创始人无法迅速获得人力资源、财务、销售和董事会管理方面的技能，再由创始人管理会让一家公司迅速消失。所以他们聘用职业经理人。这些新 CEO 也要起到及时消除创始人过激脾气的作用。
在上个世纪，技术投资人认识到职业化 CEO 们在扩张阶段非常有用，而在寻找产品定位阶段没用 技术周期变成一个跑步机，存活下的初创公司需要在不断创新的周期中。这种初创文化一直持续多年–谁做得最好呢？当然是那些创始人。
创始人习惯于嘈杂与混乱。与之相对，职业经理人试图让嘈杂恢复秩序并且经常杀死初创文化。创业公司认识到 指导创始 CEO 壮大公司比让职业经理人 CEO 找到新的创新机遇更容易。他们是对的。我在我访问过的公司中确实如此。-过去 5 年中已经出现过 200 个独角兽公司，大多数仍人是由创始人掌管。
因此，初创公司发现并且相信，有一个对创始人友好的董事会，且公司继续由创始 CEO 经营，公司会增长得更快。创始人的现实扭曲力吸引了大量有同样愿景的雇员。这是非常引人注目的，每个人为了少量薪水和一些股票工作极长的时间。他们是幸运的，他们掌握了正确的节奏，经过几年痛苦的时间发现正确的产品或市场，然后上市。那些早期员工因为股票变现而获得了回报。
但问题在于员工超过 1000 的某个时刻，优先股的大量支出结束，随后股票在 IPO 后开始上涨。 但是 CEO 从未注意到其他公司为他支付 95%的时间已经结束。飞到他私人院子里的远程办公室，被那些身价千万美元的早期员工包围。被”要么留要么走“咒骂的却是一些新员工。
公司现在吸引那些实习生，只想在他简历上留下”在热门公司工作过的””一段经历。但是由于薪酬水平低于平均水平，他们会待到足够长的时间来完善他们的简历后为了更高的报酬而离开-通常去一家初创公司。
由于只有少量高级工程师认为是一个理想的工作场所，公司开始的技术优势已经开始恶化。

叫醒服务

创始人的哪些特点不利于运营大型公司谁也不知道，因为没有最佳实践、没有课程，也没有标准模型。创始人团队很少把初创公司变成大公司，一点也不奇怪。
再创业者通过敏捷、无情、顽强甚至不合理并成为 CEO 驱动公司增长，有时候可能，但很艰难。
这就意味着快速学习一系列新技能-重大自我升华、当通过直接汇报的方式不能掌握整个公司的时候，就需要建立可重复流程实现规模化管理。有时候这只是在电话唤醒危机之后才会发生。作为一家创业公司，创始人和董事会需要认识到，最重要的改变不是关于系统、办公环境或是硬件。他们要认识到员工才是他们最宝贵的财富
伟大公司的创始人们知道如何保持他们的激情，但也知道人性。

后记

我现在可以讲这个故事了，作为负责人离开公司，并在不同的领域创建了自己的创业公司。在接下来的 6 个月里，70 名员工中 55 人要求离开。其中 25 人加入了他的新创业公司，剩下 30 人呢？成立了 6 家创业公司。

得到的教训

  • 当你长大后，要当心那些不必要的教训
  • 认识公司规模变化时的边界
  • 认识到在小规模是驱动创新的文化在公司变大后不再适用

zabbix discovery分为网络发现（net discovery）、活动代理自动注册（Active agent auto-registration）和低级发现（low-level discovery）。网络发现可根据ping或agent或其他网络服务找到满足检查结果的主机，并执行其他动作。可执行的动作有

配置网络发现

1 配置-》自动发现 中创建发现规则

2 配置-》动作 中设置他条件

3 监控-》自动发现 中查看自动发现的结果

故障排查

配置了自动发现一直没有发现主机，先检查是否有自动发现进程

若没有，检查/etc/zabbix/zabbix-server.cfg StartDisvoer字段是否有值，或将参数注释去掉，手动设置zabbix discover进程为4，然后重启zabbix-server。

再检查web界面自动发现规则是否启用，网段是否正确。

最后检查动作中条件是否引用发现规则。

安装ansible

centos7

ubuntu

配置使用

以monitor用户连接主机hosts执行who命令
ansible -i ./hosts all -a ‘who’ –ask-pass -u monitor

错误排查

1 安装完成后找不到配置文件，无/etc/ansible目录，通过卸载pip安装的ansiblesudo pip uninstall ansible,重新使用yum安装解决yum install ansible

2 FAILED => Using a SSH password instead of a key is not possible because Host Key checking is enabled and sshpass does not support this. Please add this host's fingerprint to your known_hosts file to manage this host.
通过修改/etc/ansible/ansible.cfg 中host_key_checking = False行注释去掉。

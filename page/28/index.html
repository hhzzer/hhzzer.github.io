win10使用docker写hexo博客

发表于 2022-06-29 更新于 2022-07-01

发表于 2018-07-03
日常作业平台换为 win10 后，博客需要恢复活力。

问题

hexo 博客使用 nodejs 后端框架，win 平台上使用不方便。尝试过使用 win10 ubuntu bash，无法安装完全 npm 包。尝试使用 hyper-v 虚拟机,过程不流畅。最终选择使用 docker，挂在本地目录，在 docker 中写 blog，在宿主机查看效果。

处理过程

1 安装 docker 环境

  1. 操作系统为 win10 专业版，bios 开启虚拟化支持，开启 windows 功能 hyper-v。
  2. 注册 docker 账号，下载”Docker for Windows Installer.exe”百度云
  3. 验证 docker 环境，执行 docker info，能显示正确信息，表面 docker 安装成功。
  4. 修改 docker 镜像为中国地址。右键点击鲸鱼图标-settings-deamon-registry mirrors 中输入https://registry.docker-cn.com并应用。

2 测试并制作 hexo-server 的 docker 镜像

通过 docker search hexo 或在dockerhub,查看 hexo 相关的镜像的 Dockerfile，了解基本的制作过程。发现使用 node 镜像制作 hexo server 比较方便。
使用 node 镜像运行 container，运行安装 hexo 测试

3 将部署过镜像化

参考https://hub.docker.com/r/emitting/hexo/~/dockerfile/

4 使用 hexo-server 镜像运行容器写博客

1 将原 hexo 博客文件除 npm 模块目录外，复制到目录”~/myweb/blog”
2 在 powershell 中使用自定义镜像，运行容器，并初始化原博客

遇到的问题及解决

  1. docker 字符界面输入中文问题
    通过在 container 启动中加入环境变量参数”env LANG=C.UTF-8”解决。
  2. 运行容器时无法映射端口
    原因是宿主机网络切换，重启 docker-deamon 解决
  3. 无法挂载本地目录
    在 docker-deamon 中 settings-sharedvice 中勾选驱动器。可以把时区设置配置到镜像制作中

阿里云平台学习

发表于 2022-06-29 更新于 2022-07-01

表于 2018-07-04
学习阿里云的基础设施建设，为公司基础设施建设做参考。

飞天云操作系统

数据中心-linux 集群为基础物理结构，在此之上构建云。
图中黄色部分为飞天云操作系统，基于 8 大子系统构成。远程过程调用、安全管理系统、分布式协同服务器、资源管理系统、分布式文件系统、任务调度系统、集群部署系统、集群监控系统。
和传统的单机操作系统如 linux 可以类比。
操作系统的作用是管理计算机的硬件资源，提供系统调用便于用户使用计算资源。计算资源包括 cpu、内存、磁盘、网络、输入输出。操作系统控制 cpu 运算和 cpu 切换，为存储器提供编号便于使用。所有的计算机都是冯诺依曼计算机，面向存储器编程的计算机，特点是共享存储，串行执行。
linux 内核由 5 个子系统构成：进程管理、内存管理、虚拟文件系统、网络子系统机 IPC 进程间通信。
阿里飞天云操作系统中任务调度系统对应进程管理，内存管理对应资源管理系统，分布式文件系统对应虚拟文件系统，远程过程调用对应网络子系统，分布式协同服务器对应进程间通信。安全管理系统独立，便于用户管理。集群部署和集群监控独立，是辅助系统。
阿里云飞天操作系统面向的对象是安装好 linux 操作系统，配置好网络的 x86 服务器。在此之上再做抽象层，将资源池化，然后再通过软件定义计算资源、存储资源、网络资源。通过服务将资源组合，供应用系统使用。

运维介绍

运维对象包括：服务器、存储、数据库、网络、CDN、IDC、域名解析
发展阶段：稳定->效率->价值
稳定通过基础设施、容灾架构、分工界面、多套环境、容灾演练。
效率：发布排期、配管中西；故障相应指挥中心；交付周期成本预算
价值：组织协同、预案演练、容量预估。
担当的角色有对接层、组织者、优化师。

阿里云存储

基于飞天系统，实现盘古分布式存储，在分布式存储上提供多种存储类型。块存储、对象存储、文件存储、表格存储、连接服务。不同存储类型用于不同场景。

对象存储特点：海量，EB 级别存储容量无上限，万亿级别，文件数量无上限；任意大小，0-48.8TB。高可用、高可靠。安全，账号授权、加密传输存储。跨平台，RESTFul 协议。

阿里云容器服务

阿里云容器服务帮助企业实现专有云架构，有方案，有经验，给钱就行。主要方式是部署飞天敏捷版容器服务，将企业应用容器化。用钱铺平容器化过程中的坑。

其他感想

没有不能解决的架构难题，如果有，就再抽象一层。由传统数据中心架构转移到云计算中心架构，分布式系统知识不可或缺。参考资料《在线》 by 王坚

windows2008R2安装powercli

发表于 2022-06-29 更新于 2022-07-01

发表于 2018-07-06
今天需要在新环境上批量创建虚拟机，安装脚本执行环境时踩坑无数，记录备查。

问题

新环境需要批量分配 vmware 虚拟机，采用 vmware 的 powercli 工具，调用脚本创建。安装 powercli 工具时提示依赖.net4.5 和 powershell3.0，手动安装.net4.0 和 powershell3.0 的更新包报错不支持。

解决过程

.net4.5 安装包由其他地方拷贝过来，安装时报错。环境开启外网，直接安装 vmare-powercli6.5,会自动下载.net4.5 并安装成功。自动下载安装 powershell3.0 时报错。
根据报错浏览到微软官网 powershell3.0 安装的补丁页面，下载合适的版本后，手动安装，提示不支持当前系统。仔细看先决条件，发现只能安装在 windows2008 R2 sp1 上，操作环境系统没有 sp1 补丁。下载 sp1 补丁安装。dotnet4.5 手动安装失败也是由于 sp1 补丁没安装的原因。
sp1 补丁安装完成后，再安装包含 powershell3.0 的补丁，然后再安装 VMware-powercli 工具，安装成功。
其他
管理员权限打开 vmare-powerCLI，执行命令 Set-ExecutionPolicy RemoteSigned,允许 powerCLI 执行远程命令。
查看 powershell 版本: 在 powershell 中输入 $host
查看是否有 sp1 补丁： 右键计算机-属性-windows 版本，有 sp1 会有标注，没有则空白。

教训

  1. 清楚当前操作环境，windows 版本带 sp 和不带 sp 的差别大。
  2. 下载和安装软件看清楚依赖和安装说明，了解软件需要安装的环境，下载正确对应环境的软件。

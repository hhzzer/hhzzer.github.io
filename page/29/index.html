网络架构分享总结

发表于 2022-06-29 更新于 2022-07-01

发表于 2018-07-11
网络整体架构为外层防火墙接互联网，外墙上划分 3 个区域，生产、测试、OA。外墙南向接核心交换机，核心交换机旁挂内墙。内墙上起网段网关。核心交换机南向接汇聚交换机，汇聚交换机接接入交换机或刀箱交换机。接入交换机连接物理服务器。

术语

EoR：（End of Row）接入交换机集中安装在一列机柜端部的机柜内，通过水平缆线以永久链路方式连接设备柜内的主机/服务器/小型机设备。EoR 对设备机柜需要敷设大量的水平缆线连接到交换机。
ToR：（Top of Rack）接入方式就是在服务器机柜的最上面安装接入交换机。
IRF 是 Intelligent Resilient Framework 的简称，即智能弹性架构，是 H3C 公司专有的设备虚拟化技术，将实际物理设备虚拟化为逻辑设备供用户使用。目前的 IRF2.0 是一种将多个设备虚拟为单一设备使用的通用虚拟化技术，此技术已经应用于高、中、低端多个系列的交换机设备，通过 IRF2.0 技术形成的虚拟设备具有更高的扩展性、可靠性及性能。
leaf-spine:叶脊拓扑网络，通过增加一层 平行于主干纵向网络结构的横向网络。

Overlay，顾名思义，上层的，或者说，业务层面的、用户层面的，overlay traffic 是指用户业务流量。
underlay，顾名思义，下层的，或者说，基础架构层，专门用于承载用户流量的传统的 IP 网络，只要可以提供 IP 包的转发即可。
underlay 是底层网络，负责互联互通。overlay 基于隧道技术实现，overlay 流量需要跑在 underlay 上。
SDN

其他

交换网络设备有两个功能，1 是流量转发，2 是控制流量流向。SDN 即软件定义网络，目的是将所有网络交换设备的控制功能统一，设备只负责转发流量。需要将所有设备配置 ip，像服务器一样管理设备，通过软件能力，组成一个分布式的交换网络。实现控制统一，便于网络自动化运维。

问题

结论

不能通过为 ubuntu1404 server 版指定高版本源使用 apt 升级 openssh-server。目前实践成功的方法是手动编译安装。参考

解决过程

ubuntu 14.04 desktop 版本在 2018 年前可以执行成功。
服务器安装 Ubuntu 14.04.5 server 版本后不能执行成功，报错’InRelease 错误’。到源服务器查看源内容，发现没有‘InRelease’相关文件。对比网易源，17 版本的主要软件都更新到‘artful’版本中。
Ubuntu 14.04.1 server 无法升级到 openssh-server 7.2

报错总结

1 2018 年能用 zesty 源安装 openssh-server，之后不行，报错’InRelease 错误’。原因是网易官方将 17 版本源相关软件放到‘artful’版本。更新内网源后不报错’InRelease 错误’。
2 ubuntu14.04 桌面版能升级安装 18，17 版本的 openssh-server，ubuntu14.04server 版不能升级安装。报错

原因是 1404 server 版中两个基础包 libc-bin,libc6 和系统版本有强依赖，依赖<2.20,2,26 版本。1404 desktop 版本 libc-bin,libc6 依赖版本较高>2.20.
通过使用 1404 的源，降级安装 libc 可以将安装 openssh-server 失败后，无法再安装其他软件的故障解决

在线-读书笔记

发表于 2022-06-29

发表于 2018-07-27
leader 推荐王坚博士的《在线》一书，读完写下摘要和感受。
全书分为 10 章，主要由媒体访谈内容组成。王坚博士在媒体访谈中的解答可以用 5W2H 梳理。整本书的内容和标题很相符，就是说明：在这个万物互联的时代，数据很有价值，只有在线的数据，才能发挥出数据的价值。

Why 为什么有阿里云

对标国际互联网公司，国外有亚马逊和谷歌，国内却没有规模的大公有云计算公司。GWF 和国家信息安全政策环境下，国外云公司很难进入中国。在这片蓝海领域，阿里做公有云计算，做成了类似于水电气的通用资源提供商，可以行成垄断，获利丰厚，很有商业价值。
公有云进入门槛高，阿里有技术和资金，应该可以成功。

When 什么时机做阿里云

2006 年亚马逊开始提供公有云，2008 年阿里开始做。

What 什么是阿里云

阿里云是资源公司。提供计算能力、存储能力、网络能力。接上就能用，而且价格便宜。

Which

Where 阿里云的范围

和电能与电器类比，阿里只做发电机和电力传输网络，负责把资源运送到普通人手里。这些计算、存储、网络能力能怎么用，在什么场景下发挥价值，就是电器厂家的价值所在。

How 怎么做的

从 0 做起。做分部署存储系统-盘古，做分布式资源调度系统-女娲，做资源诊断系统-华佗，做钟馗。。。形成一个由 5000 台物理服务器组成的网络操作系统-飞天。

How Much 花费

前期基础技术研发费用高，一旦研发成功，收益丰富。资源用户可以很便宜的获取资源。
计算是在线的，不是私有网络的几个机架的服务器。
数据比功能重要，功能很快就能通过技术被模仿，用户需要的是数据支撑的服务。
做 app 是在别人的花园里弄盆栽,所以要做 yunos。

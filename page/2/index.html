kubernetes基本概念

发表于 2022-07-06

发表于 2018-02-13

Kubernetes 核心组件

架构图

  • etcd 分布式数据库，保存整个集群状态；
  • apiserver 提供资源操作的唯一入口，提供认证、授权、访问控制、api 注册和发现等机制；
  • controller manager 负责维护集群的状态，如故障检测、自动扩展、滚动更新等；
  • scheduler 负责资源的调度，按照预定的调度策略将 pod 调度到相应的 node 上；
  • kubelet，node 上的管理组件，负责维持容器生命周期，负责存储卷 Volume（CVI）和网络（CNI）管理；
  • container runtime 负责镜像管理及 pod 和容器的真正运行（CRI）；
  • kube-proxy 负责为 service 低通内部的服务发现和负载均衡；
    其他推荐的插件：
  • kube-dns 负责为整个集群提供 DNS 服务
  • Ingress Controller 为服务提供外网入口
  • Heapster 提供资源监控
  • Dashboard 提供 GUI
  • Federation 提供跨可用区集群
  • Fluentd-elasticsearch 提供集群日志采集存储与查询

核心技术概念和 API 对象

API 对象是 K8s 集群中的管理操作单元。每个 API 对象有 3 大类属性：元数据 metadata、规范 spec 和状态 status。元数据是用来标识 API 对象的，每个对象都至少有 3 个元数据：namespace、name、uid；此外还有各种各样的标签 labels 用来标识和匹配不通的对象。规范 spec 描述了用户期望 K8s 集群中的分布式系统达到的理想状态，如设置 pod 副本数为 3。status 描述了系统当前达到的状态，如果与 spec 中的描述不一致，系统自动执行相关的动作，争取达到 spec 中的要求。

node

node 为提供计算能力的节点，可以是虚拟机和物理机，上面运行了 kubelet 管理节点上运行的容器，与管理节点 APIserver 通信。默认情况下，kubelet 在启动时会向 master 注册自己，并创建 node 资源。
node 包括下面状态：
地址：hostname，外网 ip，内网 ip。状态：outiofdisk，memorypressure。容量及基本信息。

名字空间（namespace）

为集群提供虚拟机隔离作用。常用来隔离不同用户。pod、service、RC、Deployment 都属于某个 namespace，node、PV、namespace 不属于任何 namespace。

User Account 和 Service Account

用户账户是为人提供账户标识，服务账户为计算机进程和 pod 提供账户表示。用户账户是跨 namespace 的服务账户不跨 namespace

RBAC 授权访问

基于角色的访问控制（Role-based Access Control），引入角色和角色绑定概念。与基于属性的访问控制（Attribute-based Access Control）相比，ABAC 访问策略智能跟用户直接关联。RBAC 中可以跟某个角色关联，用户跟一个和多个角色关联。

pod

pod 是在 K8s 集群中部署应用或服务的最小单元，支持多个容器在一个 pod 中。pod 是豆荚，容器是豌豆。同一个 pod 中共享网络和文件系统。有四种类型 pod 执行不通类型的任务：长期伺服行（long-running）、批处理行（batch）、节点后台支撑型（node-daemon）和有状态型；分别对应 Deployment、Job、DaemonSet 和 StatefulSet。
pod：豆荚，kubnernetes 管理的基本对象。pod 是篮子，容器是篮子里的鸡蛋。pod 里的容器共享一部分资源，包括 network namespace，volume。
ip 地址分配给 pod。
标签 labels 标记在 pod 上。
labels 由键/值对构成，可在一维里灵活组合筛选。

Delployment

无状态，长期运行的一组 pod 集。可以更新，滚动升级，回滚等。

job

有始有终的任务，任务完成后就退出。可以定制不通的 spec.completions 策略而不通：单 pod 型任务有一个 pod 成功就标识完成；定数成功型任务保证有 N 个任务全部成功；工作队列型任务根据应用确认的全局成功标识。

后台支撑服务集（DaemonSet）

每个 node 上都需要运行的 pod，包括存储、日志和监控等。

有状态服务集（StatefulSet）

RC 和 SC 主要控制提供无状态的服务，期控制的 pod 名字是随机的，一个 pod 故障了就被丢弃，在另一个地方重启一个新的 pod，名字会改变，关心的是 pod 的总数。一般不挂载存储。
StatefulSet 中每个 pod 名字是事先确认的，如果出现故障，会再其他地方重启一个相同名字的 pod。一般挂载有独立的存储。适合于数据库服务 MySQL、PostgreSQL，集群化管理服务 Zookeeper，Etcd 等有状态的服务。

复制控制器（Replication Controller，RC）

通过监控运行中的 pod 来保证集群中运行指定数目的 pod。

副本集（Replica set ，RS）

RS 是新一代的 RC，支持更多类型的匹配模式。通常作为 Delployment 的理想状态参数使用。

服务（Service）

客户端需要访问的服务就是 Service 对象。一个 pod 只是一个运行服务的实例，多个实例构成一个高可用的服务。高可用由每个节点上的 kube-proxy 提供。

存储卷（Volume）

docker 的存储卷作用范围为一个容器，K8s 的存储卷和生命周期作用范围是一个 pod。每个 pod 中声明的存储卷由 pod 中所有容器共享。

持久存储卷（Persistent Volume，PV）和持久存储卷声明（Persistent Volume Claim ，PVC）

PV 和 PVC 使得 K8s 集群具备了存储的逻辑抽象能力。PV 和 node 类是，是资源提供者，由集群管理员配置。PVC 和 pod 是资源使用者，根据业务的需求变化而变化。

发表于 2018-02-06
在微信读书上看到自己书架上有这本书，不知是出于什么原因加入的，通勤时间看，有意思。

带着问题去读书

在某个瞬间，有时是工作到力竭，有时是玩游戏到空虚，就会问自己：我到底为何而活？博客说明中写了“做有钱有钱的人”，但为什么呢？
有钱，让我有更多选择。生活中多数目标需要金钱支持，为了让我在选择一种自己想要的生活体验时，不会因为钱而退缩，所以要有钱。好选择源自好多选择，钱多，选择多。
有趣，因为一生短暂。存活时间只有 2 万多天，一天天都跟随人潮惯性活动，不觉得有点浪费吗？生命账户余额 2 万，每天花一块，花完就死。所以要主动花，花在让我们自己高兴的地方。有趣，会让我慧心一笑。
但是更多钱，带来等多选择时，选择本身也会带来痛苦。你愿意承受进行选择的痛苦还是接受默认选项的痛苦呢？
我选前者，自己选择时自我自由意志的体现，让我感受到我的存在。
但是有趣，真是评判时间投资的最佳选择吗？是我的价值判断标准吗？小时后认为一株稻草有趣，叶子中间宽两端细，有倒刺割手会疼，拿着它转它就转，拿来逗猫猫也跟着跑。现在认识到稻草是禾本科植物，叶为水滴状左右对称有锯齿，小猫跟着稻草跑是捕猎前的尝试。我不再认为稻草有趣了。我的有趣价值观随着我对这世界的认识不断变化。标尺忽长忽短，做选择时哪个答案时我的正确选项呢？
我迷茫了，我要在这本书中找答案，想知道历史上的智者对生活意义的回答以及他们的价值判断标尺。

如果一个人不断的问问题，还要拉着别人不放让他一直回答，直到这个人崩溃，我们可以形容这个人“苏格拉底附体”。苏格拉底他妈是助产士，他是思想的助产士。芝诺说知识就是一个圆，知道得越多圆外接触面越大，越觉得无知。苏格拉底知道很多，怀疑更多。我怀疑故我在，是他生活的目的。

投资中最简单的事-读书笔记

发表于 2022-07-06

发表于 2018-01-21
在微信阅读上花两周时间看完。
全书氛围 4 个大块讲述了投资中最简单的事是什么？如何用最简单的是投资。
投资理念
价值投资、成长投资、趋势投资
投资方法
估值、品质、时机
投资风险
投资策略
投资心理学
全书是作者从业 10 多年来价值投资的心得和经验，主要论述“买入被市场低估的股票，在市场高估时卖出”。估值、品质、时机。
把客户的钱当作自己的钱，以在一级市场投资实业的方式投资二级股票市场。
如何判断一家公司的估值高低？实际价值是牵狗的人，估值是那条狗。估值在实际价值周围波动。估值是公司当前的总市值，价值是当前公司所有的有形和无形资产。市盈率是根据当前的盈利能力，预估多少年能达到市值。
如何判断是不是一门好生意？不同行业有不同的判断指标。制造业业通过规模优势降低成本盈利，餐饮行业通过高回头率、翻台率提升单位面积带来的利润获得盈利。连锁行业复制成本、回款率。杠杆行业控制风险。波特五力分析：定价权、护城河、垄断。
时机要淡化，因为很难判断。观察周期规律可以大概率判断趋势。

超级个体44-48周

发表于 2022-07-06

发表于 2018-01-21
笔记本放家里不利于写博客
1 月 16 日听完 5 篇文章，1）如何利用别人的复制求偶策略在求偶中取胜，2）在当前应该选择生娃还是选择不生娃，3）职业转型策略，从军官到互联网医疗公司合伙人，4）开启生命可能-gap 一下，5）开启生命可能-读个书吧，6）开启生命可能-极简主义，7）开启生命可能-手作。
求偶策略有探索式和拷贝式。探索模式是指在生活中发现自己是怎么样的，喜欢怎样的状态，找到和自己类型相似的人，在一起沟通成本低，心有灵犀一点通，幸福感高。拷贝式是指不知道自己的喜好，看见别人在追求某人，自己也从众的去追求，拷贝别人的判断。利用大家的从众心里，可以制造自己被多人追的氛围，提高自己的吸引力。或者跳出目前赛道，到新的地方开启新赛道，重新估值。
不是生与不生的判断题，而是选择现在还是过一段时间生的选择题。孩子在经济学来看，对于不同收入的家庭性质不一样。中产以上家庭把孩子定义为消费品，在拥有和培养过程中获得成就感和满足感就值会购买时的成本。低收入家庭将孩子作为耐用生产品，期望孩子长大后产生更多回报。当前的投入也是为了以后的回报。分析收益损失表，可以让人做出更清晰的选择。由于损失厌恶的原因，主要说损失。生的损失：金钱、精力投入增加，打破工作连续性发展。不生的损失：双方长辈的期望，再成长一次的机会，一段不一样的生命旅程。结论：当前可以选择生，期限 1 年内。
职业转型小步快跑。利用原平台的资源提升能力，结交新的人脉，获取市场消息和能力反馈。
gap 间隔年。离开熟悉的环境，在新的环境使用新的坐标系，然后审视之前的坐标系选择及定位。不识庐山真面目，只缘生在此山中。
读个书是一种投资，可能是收益率最高的一种方式。前提是有目标和行动路径。如果没有目标，所有的努力都是在做摩尔运动。
极简主义是重新审视知己需求的过程。通过树立自己的需求，认识自己的喜好偏见。假如我明天就会失去光明、双手、双腿、金钱、房子、老婆，我今天该怎么度过？哪些是我失去以后会最后悔的？这类思维游戏和极简生活是一个道理。
手作，通过触感感受这个世界，听内心对世界的判断。我们的视觉、听觉等受到商家的大量广告、商品信息和文化灌输，产生的判断可能与内心价值有较大偏差。手的触觉这种最原始的最强大的感受力还没被污染，苏醒把，触感巨人。

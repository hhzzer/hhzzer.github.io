python模拟登录浪潮BMC

发表于 2022-06-20 更新于 2022-06-29

背景

服务器带外密码经常不正确，需要有一个工具去巡检服务器带外的登录信息是否准确，考虑使用 python requests 去模拟登录。

页面分析

浏览器设置

chrome 浏览器访问带外地址，F12 开启调试模式，勾选“Preserve log”，记录跳转日志。

输入错误的密码，观察页面跳转

输入错误密码后，发现有 3 个请求页面
randomtag 页面，使用 get 方法，返回一个随机 tag


红色 session 页面
使用 delete 方法，删除存在的 session，删除失败。

第二个 session
post 请求，ajax 请求。post 数据中需要 4 个字段，用户名、密码加密过，login_tag 为 randomtag 请求的返回值。
response 内容为认证信息，其中的 CSRFToken 字段需要注意。



输入正确登录信息，观察页面

最开始依然有 3 个请求页面，user 页面应该是登录成功后的页面。

user 页面为 get 请求，请求头包含了 CSRFToken.返回值看不了，可以看看 user 后面的页面。

调试工具选择 Fetch/XHR,排除 js，png 的请求。

查看 chassis-status 请求，get 请求，带 CSRFTokent，返回 json 格式文本。登录成功后可以请求这个页面，能获取期望的值，表明登录成功，获取失败，表明登录失败。

加密数据分析

查看 session 页面提交的 post 数据，发现加密的用户名和密码，多次输入相同的内容，密文不变。表示加密和时间无关。
观察输入框页面源码，页面 submit 提交按钮点击后，执行 doValidate()函数。

doValidate（）函数在 signin.js 中，包含了 login 的 ajax 请求函数。

查看 js 文件，有 signin.js Blowfish.js 两个文件。

signin.js 中有加密的函数。Blowfish.js 是一个加密算法，signin 应该引用了加密算法。算法看不懂。

要完全解密，手动制作密码表，把数字，大小写字母，特殊字符输入到用户名输入框，得到对应关系。
也可以把有的用户名，密码都在输入框中加密，获得对应关系，直接在程序中做转换。

《软技能》读书笔记1-事业

发表于 2022-06-10 更新于 2022-06-29

全书从人的角度，解答软件开发人员关于事业、思想、身体、精神四个方面的问题。
事业从职业、自我营销两个角度谈。思想从学习、生产力、理财三个角度看。身体=健身，精神=形而上。

职业

你所能犯的最大错误就是相信自己是在为别人工作。这样以来你对工作的安全感已然尽失。职业发展的驱动力一定是来自个体本身。记住：工作是属于公司的，而职业生涯却是属于你自己的。 –厄尔.南丁格尔

决不要做他人都在做的事

从商业上来看，稀缺产生价值。做软件开发也好，做运维工作也好，第一重要的是把工作作为一桩生意来看待。软件开发人员是出售自己把想法变成软件产品的能力，运维人员是向业务系统买保险，兼顾理财咨询。
没有明确的方向，你的每一步都是徒劳。
你希望从你的职业生涯中收获什么？你希望自己 5 年或者 10 年后在哪里？
先设置清晰的不那么具体的大目标，用来确定自己是离目标更远还是更近。
较小的目标可以让你在航行在自己的轨道上，激励你保持航向朝着更大的目标前进。
追踪你的目标，在必要时做调整。定时机制来检查。
采取行动：把你的大目标写在自己每天必经之地，每日三省吾身–我在追求什么。

人际交往能力远比你想象的重要

在软件开发领域，我们大多数时候是与人，而非与计算机打交到。我们所写的代码首先是供人阅读，其次才是让计算机执行。
每个人都希望感到自己很重要。永远不要批评。换位思考，停止用”我想要什么“，考虑什么才是他们需要的。避免争吵，既然逻辑和纯粹的理性无助于说服一个尖声大叫的小孩明白睡觉能让他得到充分的休息，那么，你也不可依此方法来说服一位备受轻视的同事接受“我做事的方式才是最好的”。
如果有人不管怎样都无法相处，总是贬低别人，对生活消极，不要试图改变他，要远离他。
采取行动：1）跟踪你一天的与人打交道的每件事，2）阅读人性的弱点。

破解面试之道

功夫在面试之外，博客，与面试官熟悉。展现当前能做什么和未来的潜力。
采取行动：列出你想去的公司，去认识人。

雇员不是唯一的选择

雇员、独立咨询师、创业者，你有 3 种选择。
采取行动：列出你认识或听说过的三种类型的人员。

专业化很重要

虽然我很喜欢海因莱因的下面这段话，但是在聘请律师时还是要找专业的。生活要有各种体验，要从工作，或者职业中交换价值，你必须专业化。

一个人应该能换尿布、杀猪、开船、设计房子、写十四行诗、砌墙、接脱臼的骨头、救濒死的人、服从命令、发送命令、接受合作、独立行动、解数学方程、分析新的问题、铲粪、电脑编程、做出可口的饭菜、善于打架、勇敢的死去，只有昆虫才专业化！–海因莱因

专业化的规则是：专业化程度越深，潜在的机会就越少，但获得这些机会的可能性越大。

net.ipv4.tcp_sack 导致网络访问慢

发表于 2021-04-15 更新于 2022-06-29

背景

在阿里云边缘节点开通的 ECS，操作系统 centos，使用 yum 下载安装软件很慢。

解决过程

阿里专家通过抓包排查，发现内核参数有问题，建议配置： net.ipv4.tcp_sack =1

原因

使用 Selective ACK﹐它可以用来查找特定的遗失的数据报— 因此有助于快速恢复状态。该文件表示是否启用有选择的应答（Selective Acknowledgment），这可以通过有选择地应答乱序接收到的报文来提高性能（这样可以让发送者只发送丢失的报文段）。(对于广域网通信来说这个选项应该启用，但是这会增加对 CPU 的占用。)

《计算机程序的构造和解释》

发表于 2021-04-15 更新于 2022-06-29

多次拿起这本书，好多示例看不懂，记录一下我理解的，有趣的。后续尝试通过视频学习看看。

教师，长官是规划者，学生，士兵是被规划者。计算机程序的设计，就是用人的大脑，去规划程序的运行，解决实际问题。规划是复杂的，因为很多数据在运行过程中产生。怎么去解决规划的复杂性，就是本书重点要讲解的。

本书关注的主要对象是：人的大脑，计算机程序的集合以及计算机本身。
解决复杂性的方法：

  1. 通过创建抽象去控制复杂性，包括对过程的抽象和对数据的抽象。（第 1、2 章）
  2. 通过建立约定界面，用对象或过程的方式去组合一些标准的片段，解决并发等问题。（第 3 章）
  3. 创建新的语言，对语言进行抽象（第 4 章）

里面讲的一些概念，如：正则序，应用序，很重要，慢慢理解。

金句摘录

每一个计算机程序都是现实中或者精神中的某个过程的一个模型，通过人的头脑孵化出来。

如果说艺术解释了我们的梦想，那么计算机就是以程序的名义执行着他们。

一个计算机语言并不仅仅是让计算机去执行操作的一种方法，更重要的，他是一种表述有关方法学的思想的媒介。因此，程序必须写的能够供人们阅读，偶尔地去供计算机执行。

真正的问题：弄清楚我们需要去计算什么，怎样将问题分解为一组可以控制的部分，如何对这样的部分开展工作。

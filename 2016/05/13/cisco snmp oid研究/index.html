cisco snmp oid研究

发表于 2016-05-13 更新于 2017-06-16

最近使用zabbix监控cisco交换机相关情况，计划使用snmp协议获取监控指标数据，一边研究一边记录整理。
##配置snmp协议
首先需要在交换机配置snmp服务,其中xxx为自定义连接字符串，RO为只读。
hostname(config)#snmp-server community xxx RO
配置完成snmp协议后，就可以通过命令或工具，附带OID参数，获取感兴趣的数据。
##查询snmp信息方法
linux中可以通过snmpwalk 查询snmp信息。也可以通过Getif工具在window下查询，运行时注意使用兼容模式运行。

了解了配置snmp服务和查询方法后，要获取监控指标数据，关键是找到OID（Object Identifiters）,也称对象标识符。OID相当于通过snmp协议释放出大量数据组成数据库中的查询条件，输入不同OID，获得不同结果。上面例子中OID=’SNMPv2-SMI::enterprises.9.3.6.3.0’,就查询出设备序列号。

##获取OID方法

  1. 通过cisco官方网站查询
  2. 通过下载MIB（Management Infomation Base)库，直接在本地查询。
    一般是先通过网站，查询OID结构，然后再通过Getif工具查询设备，观察返回数据。

###常用OID整理

MIB和OID
OID(对象标识符），是SNMP代理提供的具有唯一标识的键值。MIB（管理信息基）提供数字化OID到可读文本的映射。
iReasoning MIB Brower 使用方法
下载全平台包link,解压后cmd到 mibbrowser\ireasoning\mibbrowser\lib 目录，运行 java -jar brower.jar 运行软件。
下载 cisco MIB包link,使用MIBbroswer加载包后执行。

Ubuntu-wine折腾记

发表于 2016-11-23 更新于 2017-06-16

事件起因于想在ubuntu系统上连接MS sqlserver 数据库。之前在windows环境使用了Navicat的产品，连接MS server很方便。昨天经过一番查询，发现可以在ubuntu上运行Navicate prumu连接各种类型数据库，就下载下来运行。运行前提是要有wine，就sudo apt-get install wine,安装了wine。一开始下载的是中文版，运行后窗口中文显示不出来，就下载安装了英文的。结果安装后可以连接MySQL,ProgresSQL,Oracle,SQLite,MarriaDB就是不能连接MS sqlserver。之后还找了squirrel 在java上运行的数据库连接平台，由于驱动没找到也放弃。后面使用wine安装了Navicate for sqlserver,能成功安装但无法运行。终于放弃，在虚拟机上安装了。

知道了wine之后，想到自己的邮件大部分还在foxmail上未转移，要是能用wine使用foxmail多好。随即将保存的foxmail安装程序和数据文件一起拷贝来，使用wine运行，居然能运行，喜出望外。通过转发的方式将thunderbird上已发邮件转移到foxmail。本以为能开始愉快的使用foxmail了，结果发现两个致命问题：1 发邮件无法输入联系人信息。 2 搜索无法输入中文。到处找方法都没解决，就算了，以后foxmail就用来查询历史邮件吧。

经过这些折腾，用ubuntu工作，需要面对的问题
1 硬盘怎么规划?使用双系统还是虚拟机，虚拟机安装什么系统，虚拟磁盘空间多大？
硬盘剩余30G空间，空着。其余空间分配给ubuntu系统。ubuntu系统安装分区根据鸟哥建议分。虚拟硬盘使用30G，安装windows xp系统。
2 基本软件有哪些，哪些用linux版本替代，哪些使用wine安装，哪些使用虚拟机。

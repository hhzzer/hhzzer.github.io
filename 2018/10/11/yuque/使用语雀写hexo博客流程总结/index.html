使用语雀写hexo博客流程总结

发表于 2018-10-11 更新于 2022-06-29

背景

hexo 博客很好，但是没有一个好的编辑器。markdown 编辑器用了一大堆，最后还是在 vim 上写。写一般的文档还行，要是加图，加附件，加表格就很反人性了。
语雀是一个好的文档仓库，编辑器所见所得，支持 markdown 的快速命令格式，是我用过的最好的编辑器。
用语雀编写存储文档，并用 hexo 博客发布，众享丝滑。

实践

  • 基础环境

使用 win10 笔记本+virtualbox 安装 centos7 ，网卡 NAT 模式作为 hexo 博客运行环境。使用 vscode-Remote Workspace 插件通过 vsftp 的方式连接到 centos7 虚拟机。虚拟机执行命令使用 git 的 bash，ssh 到虚拟机中执行。

  • 运行环境配置

  1. 安装 nodejs，版本>8.0，参考

  2. 安装 hexo 博客工具并初始化博客

  3. 安装 yuque-hexo 插件并配置语雀知识库（用于写博客的知识库）

  • 写博客流程

  1. 在语雀博客知识库中写文章并发布

  2. ssh 到 hexo 运行环境，执行
    YUQUE_TOKEN=xxx yuque-hexo sync  同步下载语雀文章到博客文档目录

  3. 生成并发布博客 hexo d -g

  • 博客编写 tips

  1. 支持图片，图片存储在语雀上。

  2. 支持标签和分类，tags: [Hexo]  categories: 教程，在元信息和正文之间插入分割线即可。

总结

不管工具多好，流程多顺，重要的还是文章的内容。好工具能让你表达更顺畅，更愿意写。
目标：能持续输出有价值的东西，坚持 10 天再说。

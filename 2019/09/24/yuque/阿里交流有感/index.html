阿里交流有感

发表于 2019-09-24 更新于 2022-06-29

背景

公司之间进行技术和文化交流，了解别人在做什么，怎么做很重要。

内容

企业文化

使命、愿景、价值观。使命回答为什么，表明存在的意义。愿景是回答是什么，长成什么样子。价值观帮助做判断，在成长过程中的具体决策，远眺使命灯塔，近看价值观路标。
企业文化不考核，不和钱挂钩，永远只能挂在墙上。企业文化要形象具体，不要太过抽象，不然无法指导行为。
阿里参考

花名：打碎原来的自己，在阿里重新成长。提供谈资，拉近距离。

商业模式

数字化、网络协同、数据智能。
数据业务化，业务数据化。
结合阮一峰的周刊更能支持这一点。个人成长，也要进行数据分析和记录。有些记录可以通过系统、物联网设备记录，但是分析只能主动复盘。

github 很多人为了让绿色小方格子不要中断，就会尽量每天提交代码。时间一长，真的多做了不少项目。
这件事的启示就是，不管做什么事，都应该进行数据记录和数据分析。对个人来说，可以从一个业余爱好者变成专业人士；对产品来说，可以从一个普通产品变成数据产品。

阿里云、IOT，成为被集成者（质疑，被集成者利润低。）
